# DESIGN.md — 동해시 AI 아이디어톤 키오스크 프로토타입

## 플랫폼 확정

**데스크톱 고정형 단일 화면. 반응형 아님.**

coding 스킬의 표준 A형(앱 모바일 고정)/B형(반응형 웹) 어느 쪽에도 해당하지 않는 제3의 케이스로 별도 정의한다.

- 뷰포트: **1600 x 1000px 고정**
- 화면 전체를 하나의 캔버스로 취급. 브레이크포인트 없음. 320px~2560px 대응 불필요.
- 웹에서 열릴 때: 1600x1000 고정 캔버스를 **scale-to-fit**으로 창에 맞춘다. 반응형 리플로우가 아니라 비율 유지 축소만 한다(`min(innerWidth/1600, innerHeight/1000)` 배율, `transform-origin: top center`, 남는 여백은 `kiosk-black-frame`).
  - **scale() 예외**: AGENTS.md CHECKLIST의 "scale 금지"는 개별 UI 요소(버튼 hover 등)의 scale 변형을 막는 취지다. scale 사용은 다음 **세 곳만** 예외로 허용한다: ① 최상위 뷰포트 scale-to-fit, ② 키오스크 카메라 줌인/줌아웃 연출(KioskCamera, PROMPT 06), ③ 종이 수령 연출(PaperReceiveOverlay, FIX-D — 클릭 시 종이를 좌측 영역 중앙으로 이동+1.8배 확대→유지→소멸). 개별 버튼/카드에는 여전히 scale을 쓰지 않는다.

## 레이아웃 그리드

```
┌─────────────────────────┬─────────────────────────┐
│                          │  우측 상단 (800 x 500)   │
│                          │  대시보드 (현황 목록)     │
│  좌측 (800 x 1000)       ├─────────────────────────┤
│  키오스크 원본 재현       │  우측 하단 (800 x 500)   │
│                          │  AI 판단 및 분석 카드     │
└─────────────────────────┴─────────────────────────┘
```

- **좌우 65:35 분할(PROMPT 05)**: 좌측 키오스크 기기 **1040px**, 우측 대시보드/AI **560px**. (tokens.js `layout.leftPanelWidth` 1040 / `rightPanelWidth` 560)
- 좌측: 회색 금속 기기 몸체 안에 스크린 컬럼(~70%, 검정 베젤 + 3:4 스크린)과 하드웨어 컬럼(~30%, 결제모듈·지문인식기 등)을 나란히 배치. 기기는 PROMPT 06에서 하단부/줌이 붙을 세로로 긴 컨테이너.
- 우측: 560px 고정폭. 상단 대시보드 600 / 하단 AI 분석 400 (FIX-C, 500:500→600:400). 경계선 1px solid 구분선.
- 대시보드는 "시니어 교육 강사용 대시보드": 시니어 A(실제 키오스크 조작)~F(가상 시뮬레이션 5명)이 서로 다른 서비스로 시차 등장해 실시간 진행. 카드는 [라운드 사각 아바타][서비스명(+실습/완료체크)·현재단계·진입][상태 dot+텍스트][큰 경과초] 구조. **최대 6행 고정, 스크롤 없음**. A(실습) 행은 항상 정확히 1개 — 새 세션 시작 시 기존 A 행을 교체(누적 금지). 진행중=dot 깜빡임+카운트업, 완료=초록 dot+체크(경과 고정), 막힘=전환 순간 shake 1회+절제된 빨간 테두리·좌측 액센트(경과 고정). 정적 더미 시간값 없음(전원 0초부터). **막힘/완료가 되면 경과 카운트 정지**(어떤 시니어도 표시 경과 2분 초과 없음).
- **대시보드 카드 톤(FIX-D, AI티 제거)**: 기본 카드는 전부 흰 배경 + 1px `dash-border` + radius 10px 통일(완료 연초록 틴트 폐기). 막힘 카드만 예외 강조하되 톤 다운(흰 배경 유지 + 1.5px 빨간 테두리 + 좌측 안쪽 3px 빨간 세로 액센트). 선택 카드는 1.5px 파란 테두리 + 옅은 파란 배경(`dash-select` #F5F8FF). 상태는 알약 채움 뱃지 대신 **8px 색 dot + 13px 700 텍스트**(진행중=파랑, 완료=초록, 막힘=빨강). 아바타는 radius 10px 라운드 사각(36px, A=파랑, B~F=`dash-avatar-bg`+`dash-text-strong`). 카드 간격 10px, 패딩 상하 14px·좌우 16px.

기기(KioskDevice)는 세로로 **캐노피("무인민원 발급창구") → 본체(스크린 컬럼 + 우측 하드웨어 컬럼: 현재금액/현금투입/노란버튼/카드/QR) → 증명서·지문확인 띠(빨간 네온 링) → 은색 받침대(영수증/안내스티커/시각장애인키보드/이어폰)** 로 구성된다. KioskCamera 줌인은 **캐노피+본체+증명서/지문 띠**까지 프레이밍(FIX-H — 지문확인이 하단 중앙으로 내려가 줌인에도 보여야 S5가 동작 시연됨), 줌아웃은 기기 전체. 검정 몸체라 줌인 시 좌우 여백은 카메라 배경(검정)과 자연스레 이어진다. **지문확인만 동작(S5 클릭→스캔→M6), 나머지 하드웨어는 장식.** 종이(CertificatePaper)는 실물 문서를 복제하지 않고 우리 졸업증명서 목업(개인정보 회색 바 + 빨간 직인)으로 만든다.

**하드웨어 리얼리즘 예외**: 좌측 기기 섀시(chassis) 컴포넌트에 한해 금속·유리 질감 표현을 위한 `linear-gradient`와 `box-shadow`(inset 음각/눌림 그림자 포함)를 허용한다. 이 예외는 하드웨어에만 적용되며, 스크린 내부 UI와 우측 대시보드/AI 패널은 여전히 flat(그림자·그라데이션 없음)을 유지한다. 하드웨어 라벨은 전부 검정 알약 + 연두 테두리 + 흰 글씨로 통일한다.

## 컴포넌트 스타일 규칙

- **좌측 키오스크 hover 금지**: 실물 키오스크엔 hover 효과가 없다. 좌측 영역 전체에서 `hover:bg-*`/`hover:border-*` 등 hover 색·그림자 변화를 쓰지 않는다. `cursor: pointer`는 실제 클릭되는 요소에만, `focus-visible` 아웃라인은 접근성상 유지. (우측 대시보드/AI 패널은 신규 설계 UI라 항목 hover/selected 강조 유지 — 예외)
- 상단 헤더바: `kiosk-header-bar`(거의 검정 회색) 배경. 레이아웃은 `[이전][다음]`을 좌측에 작게 나란히, 타이틀은 절대 중앙(흰색), 홈은 우측. 이전/다음은 연한 회색(`kiosk-gray-border`) 배경의 작은 버튼.
- **이전/다음 비활성 스타일 금지(외형·동작 분리)**: 실물 키오스크는 이전/다음이 항상 동일하게 눌리는 것처럼 보인다. 입력 조건 미충족이라고 회색으로 흐리지 않는다. "다음"은 항상 활성 외형이고 항상 클릭을 받되, 조건(주민번호 13자리·학교명 2자 이상 등)이 안 되면 클릭해도 전환하지 않는다(no-op). 조건 판정은 버튼 disabled가 아니라 화면의 `onNext` 핸들러 안에서 처리한다.
- 좌측 키오스크 버튼: 실제 사진 기준 모서리는 각짐에 가까운 둥근 사각형(radius 4px), 테두리 있는 아웃라인 스타일이 기본. S1 증명서 버튼은 줄 수와 무관하게 min-height 76px로 높이 통일.
- **S1(메인화면)만 실물처럼 검정 배경 + 흰 글자(FIX-F)**: 배경 `kiosk-main-bg`, 본문·버튼·테두리 흰색, 타이틀만 노랑. 증명서 버튼 9개는 투명 배경 + 흰 테두리 아웃라인(radius `kiosk-card` 8px). 상단 티커 바(재생/정지 + 안내 + 29/시간연장)는 `KioskFrame mainScreen` 분기로 S1에서만 렌더(다른 화면은 기존 간단 바). 하단에 무인민원발급기 워터마크(`kiosk-watermark` 흰색 80%) + 현금/카드가능(`kiosk-main-accent-blue`) + 휠체어 알약. **S3~S13 등 세부 발급 화면은 밝은 배경 유지(S1 전용 규칙).**
- 우측 대시보드 시니어 카드: radius 10px(`dash-row`), 그림자 없음(border만 사용, flat). AI 카드는 radius 8px(`dash-card`) 유지.
- 모달(2/6/13번 화면): 부모 화면 중앙에 고정폭 오버레이. 배경 반투명 딤(rgba(0,0,0,0.35)) 처리. 실제 사진에는 딤 처리가 없지만(카메라로는 안 보임) 웹 재현 시 모달임을 명확히 하기 위해 추가한다.

## 아이콘 라이브러리

- 전부 lucide-react 사용. 보조 라이브러리 사용 금지(사용자 승인 필요 규칙 그대로 유지).
- 아이콘 사이즈: 16 / 20 / 24 / 32 / 48px 다섯 단계만.
- 좌측 키오스크 화면의 지문 아이콘, 홈 아이콘, 화살표 등은 실제 사진과 최대한 유사한 lucide 아이콘으로 대체한다 (Fingerprint, Home, ChevronLeft, ChevronRight, ArrowUpDown, Printer, AlertCircle 등).

## 접근성

이번 프로토타입은 데모 목적 고정 데스크톱 화면이라 WCAG 반응형 기준은 적용하지 않지만, 색상 대비와 키보드 포커스 스타일은 기본으로 유지한다.
