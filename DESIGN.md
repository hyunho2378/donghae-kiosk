# DESIGN.md — 동해시 AI 아이디어톤 키오스크 프로토타입

## 플랫폼 확정

**데스크톱 고정형 단일 화면. 반응형 아님.**

coding 스킬의 표준 A형(앱 모바일 고정)/B형(반응형 웹) 어느 쪽에도 해당하지 않는 제3의 케이스로 별도 정의한다.

- 뷰포트: **1600 x 1000px 고정**
- 화면 전체를 하나의 캔버스로 취급. 브레이크포인트 없음. 320px~2560px 대응 불필요.
- 웹에서 열릴 때: 1600x1000 고정 캔버스를 **scale-to-fit**으로 창에 맞춘다. 반응형 리플로우가 아니라 비율 유지 축소만 한다(`min(innerWidth/1600, innerHeight/1000)` 배율, `transform-origin: top center`, 남는 여백은 `kiosk-black-frame`).
  - **scale() 예외**: AGENTS.md CHECKLIST의 "scale 금지"는 개별 UI 요소(버튼 hover 등)의 scale 변형을 막는 취지다. scale 사용은 다음 **두 곳만** 예외로 허용한다: ① 최상위 뷰포트 scale-to-fit, ② 키오스크 카메라 줌인/줌아웃 연출(KioskCamera, PROMPT 06). 개별 버튼/카드에는 여전히 scale을 쓰지 않는다.

## 레이아웃 그리드

```
┌─────────────────────────┬─────────────────────────┐
│                          │  우측 상단 (800 x 500)   │
│                          │  대시보드 (현황 목록)     │
│  좌측 (800 x 1000)       ├─────────────────────────┤
│  키오스크 원본 재현       │  우측 하단 (800 x 500)   │
│                          │  AI 판단 및 분석 카드     │
└─────────────────────────┴─────────────────────────┘
```

- **좌우 65:35 분할(PROMPT 05)**: 좌측 키오스크 기기 **1040px**, 우측 대시보드/AI **560px**. (tokens.js `layout.leftPanelWidth` 1040 / `rightPanelWidth` 560)
- 좌측: 회색 금속 기기 몸체 안에 스크린 컬럼(~70%, 검정 베젤 + 3:4 스크린)과 하드웨어 컬럼(~30%, 결제모듈·지문인식기 등)을 나란히 배치. 기기는 PROMPT 06에서 하단부/줌이 붙을 세로로 긴 컨테이너.
- 우측: 560px 고정폭. 상단 대시보드 600 / 하단 AI 분석 400 (FIX-C, 500:500→600:400). 경계선 1px solid 구분선.
- 대시보드는 "시니어 교육 강사용 대시보드": 시니어 A(실제 키오스크 조작)~F(가상 시뮬레이션 5명)이 서로 다른 서비스로 시차 등장해 실시간 진행. 카드는 [문자 아바타][서비스+현재단계][상태][큰 경과초] 구조. 진행중=뱃지 깜빡임, 완료=연초록+체크, 막힘=전환 순간 shake 1회+연빨강. 정적 더미 시간값 없음(전원 0초부터 카운트업).

기기는 상단부(스크린 + 하드웨어 컬럼, 금속 그라데이션)와 하단부(증명서 출구 / 시각장애인키보드 / 거스름돈, `bodyBottom` 차콜)로 구성된 세로로 긴 컨테이너(KioskDevice)다. KioskCamera가 이를 scale하여 S1~S12는 상단부만 보이는 줌인, S12 발급 시 기기 전체가 보이는 줌아웃으로 전환한다. 종이(CertificatePaper)는 실물 문서를 복제하지 않고 우리 졸업증명서 목업(개인정보 회색 바 + 빨간 직인)으로 만든다.

**하드웨어 리얼리즘 예외**: 좌측 기기 섀시(chassis) 컴포넌트에 한해 금속·유리 질감 표현을 위한 `linear-gradient`와 `box-shadow`(inset 음각/눌림 그림자 포함)를 허용한다. 이 예외는 하드웨어에만 적용되며, 스크린 내부 UI와 우측 대시보드/AI 패널은 여전히 flat(그림자·그라데이션 없음)을 유지한다. 하드웨어 라벨은 전부 검정 알약 + 연두 테두리 + 흰 글씨로 통일한다.

## 컴포넌트 스타일 규칙

- **좌측 키오스크 hover 금지**: 실물 키오스크엔 hover 효과가 없다. 좌측 영역 전체에서 `hover:bg-*`/`hover:border-*` 등 hover 색·그림자 변화를 쓰지 않는다. `cursor: pointer`는 실제 클릭되는 요소에만, `focus-visible` 아웃라인은 접근성상 유지. (우측 대시보드/AI 패널은 신규 설계 UI라 항목 hover/selected 강조 유지 — 예외)
- 상단 헤더바: `kiosk-header-bar`(거의 검정 회색) 배경. 레이아웃은 `[이전][다음]`을 좌측에 작게 나란히, 타이틀은 절대 중앙(흰색), 홈은 우측. 이전/다음은 연한 회색(`kiosk-gray-border`) 배경의 작은 버튼.
- **이전/다음 비활성 스타일 금지(외형·동작 분리)**: 실물 키오스크는 이전/다음이 항상 동일하게 눌리는 것처럼 보인다. 입력 조건 미충족이라고 회색으로 흐리지 않는다. "다음"은 항상 활성 외형이고 항상 클릭을 받되, 조건(주민번호 13자리·학교명 2자 이상 등)이 안 되면 클릭해도 전환하지 않는다(no-op). 조건 판정은 버튼 disabled가 아니라 화면의 `onNext` 핸들러 안에서 처리한다.
- 좌측 키오스크 버튼: 실제 사진 기준 모서리는 각짐에 가까운 둥근 사각형(radius 4px), 테두리 있는 아웃라인 스타일이 기본. S1 증명서 버튼은 줄 수와 무관하게 min-height 76px로 높이 통일.
- 우측 대시보드 카드: radius 8px, 그림자 없음(border만 사용, flat 스타일)
- 모달(2/6/13번 화면): 부모 화면 중앙에 고정폭 오버레이. 배경 반투명 딤(rgba(0,0,0,0.35)) 처리. 실제 사진에는 딤 처리가 없지만(카메라로는 안 보임) 웹 재현 시 모달임을 명확히 하기 위해 추가한다.
