# IA.md — 정보 구조 및 화면 흐름

## 전체 구조

화면은 물리적으로 하나의 페이지(단일 라우트)이며, 그 안에 세 개의 독립 상태 영역이 동시에 존재한다.

1. **키오스크 영역** (좌측): 13개 스텝 상태머신 + 3개 모달 오버레이
2. **대시보드 영역** (우측 상단): 이벤트 목록 상태(실시간)
3. **AI 패널 영역** (우측 하단): 선택된 이벤트의 AI 분석 상태

세 영역은 하나의 전역 상태(App 레벨 useReducer 또는 Context)로 연결된다. 키오스크 영역의 상태 변화가 대시보드 영역 이벤트를 생성/갱신하고, 대시보드에서 선택한 이벤트가 AI 패널에 반영된다.

## 키오스크 영역 — 13개 화면 상태

|---|---|---|---|---|

### 상태머신 다이어그램 (텍스트)

```
S1 --(증명서 선택)--> [S2 모달] --(자동)--> S3
S3 --(다음)--> S4 --(지문 클릭)--> S5 --(자동)--> [S6 모달] --(자동)--> S7
S7 --(지역 클릭)--> S8 --(다음)--> S9 --(학교 클릭)--> S10
S10 --(공개/비공개 클릭)--> S11 --(다음)--> S12
S12 --(발급 클릭)--> [S13 모달] --(자동, 3~4초)--> S1 (초기화, 재방문 로직 개입)
```

이전 버튼은 S3~S12 전 구간에서 직전 page 상태로 되돌아간다(모달 상태는 이전 버튼 대상에서 제외).

### S1 클릭 제한

S1의 9개 버튼 중 "졸업증명서(국문, 영문), 졸업예정증명서" 하나만 onClick이 실제로 S2 전환을 발생시킨다. 나머지 8개 버튼은 시각적으로 원본 사진과 동일하게 렌더링하되 onClick 핸들러를 비워두거나 아무 동작도 하지 않게 둔다. 회색 처리, 커서 변경, 비활성 스타일 등으로 구분하지 않는다. 실제 키오스크 화면에는 그런 구분이 없기 때문이다.

> **구현 메모(PROMPT 01 반영)**: 현재 코드에서는 S2 모달이 아직 없어서 SELECT_CERTIFICATE가 S1→S3로 바로 간다. S2 모달은 PROMPT 04(나머지 화면/모달 구현)에서 붙인다. sessionReducer에는 이미 DISMISS_MODAL 라우팅과 주석이 준비돼 있다고 확인됨.

## 대시보드 영역 — 이벤트 목록 상태

- 초기 상태: 예시(더미) 항목 2~3개 사전 로딩 + 실제 세션 항목 0개
- 실제 세션 항목: 키오스크가 S3에 진입하는 시점에 1개 생성, 이후 스텝 전환마다 해당 항목 갱신(화면 단계명, 진입 시각, 경과 시간, 상태)
- 상태값: `진행중` / `막힘` / `완료`
- `진행중` → `막힘` 전환 조건: 동일 화면(S3~S12)에 8초 이상 체류
- `막힘` 전환 시 자동으로 Gemini API 호출 트리거(행동 로그 전송)
- 항목 클릭 시 AI 패널에 해당 항목 분석 표시

## AI 패널 영역 — 처방 카드 상태

**실제 Gemini API를 호출하지 않는다.** 이번 프로토타입은 데모용이라 AI가 분석하는 것처럼 "보이게만" 만든다. 실제 네트워크 호출 없이 로컬 목업 함수(`mockAiClient.js`)가 정해진 딜레이 후 룰 기반 템플릿 응답을 돌려준다.

- 기본 상태: 빈 상태(안내 문구 "좌측 목록에서 항목을 선택하면 AI 분석이 표시됩니다")
- 로딩 상태: 막힘 판정 즉시 1.5~2.5초 사이 랜덤 딜레이를 주고 로딩 인디케이터("AI 분석 중...") 표시. 실제로는 계산할 게 없지만 이 딜레이가 있어야 실제 API처럼 보인다.
- 결과 상태: 어느 화면 / 경과 시간 / 추정 원인(추정형 표현) / 강사 제안 한 줄. 원인·제안 문구는 화면 ID별로 미리 작성해둔 템플릿 풀에서 선택한다(COMPONENTS.md의 `aiMockResponses.js` 참고).
- 같은 화면에서 여러 번 막혀도 매번 똑같은 문장이 나오면 티가 나므로, 화면당 2개 이상의 문구 변형을 두고 랜덤 또는 순차로 돌려쓴다.
- 두 번째 방문 시뮬레이션 시: 이전 세션에서 막혔던 스텝이 있으면 카드 상단에 "이전 방문 시 이 단계에서 어려움을 겪었습니다" 배지 추가

## 막힘 판정 로직 (졸업증명서 플로우 S3~S12, 9개 단계 전체 적용)

- 각 화면 진입 시각을 기록하고, 8초(tokens.js `stuckThreshold`) 동안 다음 단계로 전환이 없으면 해당 대시보드 항목을 `막힘` 상태로 전환
- 전환과 동시에 AI 패널 로딩 상태 트리거(위 목업 흐름 실행)
- 사용자가 8초 이후에라도 실제로 다음 단계로 넘어가면 상태는 `완료`로 갱신되고, 이미 표시된 AI 카드 내용은 그대로 유지(사라지지 않음)
- S3, S4, S5, S7, S8, S9, S10, S11, S12 전 단계 동일 로직 적용. 화면마다 다른 건 임계값이 아니라 AI가 보여주는 추정 원인 문구 내용뿐이다.

## 재방문(2회차) 시뮬레이션 로직

- S1 화면에 "다시 시작하기(2회차 시뮬레이션)" 트리거를 별도로 둔다(데모 전용 버튼, 실제 키오스크에는 없는 요소이므로 대시보드 영역 쪽에 배치, 좌측 리플리카 안에는 넣지 않는다)
- 이 트리거 클릭 시: 직전 세션에서 `막힘` 상태였던 스텝 ID를 기억한 채 S1부터 재시작
- 재시작된 세션이 해당 스텝(예: S7)에 도달하면, 키오스크 화면 자체는 원본 그대로 두되 대시보드 이벤트 항목에 "지난번 막혔던 지점" 배지를 표시하고 AI 패널에 우선 안내 문구를 노출한다
