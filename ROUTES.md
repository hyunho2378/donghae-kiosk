# ROUTES.md — 라우팅

이 프로토타입은 물리적으로 단일 화면 데모다. 페이지 이동 자체가 없고, 모든 "화면 전환"은 좌측 키오스크 영역 내부의 상태 전환(S1~S13)이다. 따라서 React Router v6는 최소한으로만 쓴다.

## 라우트 목록

| 경로 | 컴포넌트 | 설명 |
|---|---|---|
| `/` | `App.jsx` | 유일한 실질 화면. 3영역 레이아웃 전체 |

## 라우팅을 쓰지 않는 이유

coding 스킬 기본값은 React Router v6 포함이지만, 이 프로젝트는 원본 스펙("웹 고정형 단일 화면")상 여러 페이지로 나눌 이유가 없다. 다만 향후 확장(예: 결과 요약 페이지, 강사용 별도 대시보드 페이지 분리)을 고려해 React Router는 설치만 해두고 라우트는 1개만 등록한다.
