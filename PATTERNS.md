# PATTERNS.md — 반복 UI 패턴

## 카드 (대시보드/AI 패널 전용)

- 배경 `dash-surface`, 테두리 1px `dash-border`, radius 8px, 그림자 없음
- 내부 패딩 16px
- 막힘 상태 카드만 예외적으로 배경 `dash-status-stuck-bg` + 테두리 `dash-status-stuck`

## 리스트 (EventList)

- 세로 스택, 항목 간 간격 8px
- 실제/예시 구분은 회색 "예시" 배지로만 한다(실제=배지 없음, 예시=회색 배지). 좌측 세로 컬러 바는 쓰지 않는다(FIX-A 7 — 카드 단색화)
- 항목 클릭 시 배경 살짝 강조(hover와 selected 상태 구분)

## 모달 (키오스크 전용, M2/M6/M13)

- `KioskModal.jsx` 공통 wrapper 사용
- 배경 딤 처리(rgba(0,0,0,0.35)), 모달 본체는 원본 사진 레이아웃 그대로(중앙 상단 정렬)
- 버튼 없음, 전부 자동 전환. 자동 전환 딜레이는 다음 기준으로 시뮬레이션한다(실제 하드웨어 타이밍이 아니라 데모용 임의값):
  - M2 서버 접속 중: 1.5초
  - M6 본인확인 완료: 1.2초
  - M13 인쇄중: 3초

## 폼 / 키패드 (S3, S8, S11)

- 숫자 키패드: 3열 그리드, 버튼 정사각형에 가까운 비율, 실제 사진 순서(1-2-3 / 4-5-6 / 7-8-9 / 삭제-0-정정) 그대로
- 한글 키패드: 자음 5열, 모음 4열 별도 그리드로 분리(실제 사진 배치 그대로)
- "다음" 버튼은 입력 조건 미충족 시 비활성(회색), 조건 충족 시 `kiosk-blue-primary` 활성화

## 빈 상태 (AI 패널 기본)

- 중앙 정렬, 아이콘(lucide `MessageSquareDashed` 또는 유사) + 안내 문구 1줄
- 색상 `dash-text-secondary`, 장식 최소화

## 막힘 상태 강조 (EventItem)

- 8초 경과 시점에 상태 뱃지 색상 전환(`dash-status-progress` → `dash-status-stuck`)
- 카드 테두리와 배경도 동시 전환
- 전환 애니메이션은 200ms ease, 과도한 깜빡임 효과 사용 금지
